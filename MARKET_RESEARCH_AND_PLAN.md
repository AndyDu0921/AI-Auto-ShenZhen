# 深圳 AI Agent 个人创业方案（2026-04-09）

## 最终结论

最适合个人创业、低成本、可以直接启动的 2 个方向：

1. **外贸 / 跨境询盘转单 Agent**
2. **产品资料 / 工厂知识库销售助手**

原因很简单：

- 深圳跨境与制造业密度都足够高，客户好找。
- 这两个方向都能先从轻交付开始，不必一上来接 ERP / MES / 深度 CRM。
- 结果直观，老板容易看懂，Demo 容易成交。
- 两者还能共享一套底层能力，后续再自然扩展到 RFQ / BOM 解析。

---

## 五个方向的实战排序

### 第 1 名：外贸 / 跨境询盘转单 Agent
**适合度：9.5 / 10**

为什么最适合先做：

- 痛点明确：业务员漏跟、慢跟、不会筛客、英文首轮回复慢
- ROI 直接：只要多转化 1 单就值钱
- 老板易理解：询盘 -> 打分 -> 回信 -> 跟进
- MVP 容易：先接邮箱 / 官网表单 / 网站聊天 webhook 就够
- 扩展空间大：后面可加 RFQ 解析、报价草稿、CRM 推送

最小可卖版本：

- 统一接入询盘
- 自动提取型号 / 数量 / 认证 / 目标市场
- 自动生成英文首轮回复
- 自动给销售建议动作

---

### 第 2 名：产品资料 / 工厂知识库销售助手
**适合度：9.2 / 10**

为什么很适合个人创业：

- 交付最轻，最快上线
- 不依赖复杂外部系统
- 文档资料企业本来就有
- 很容易做成演示：上传手册 -> 立刻问答
- 很容易拿第一笔钱：按部门、按知识库、按站点都能卖

最小可卖版本：

- 上传 PDF / DOCX / XLSX / TXT
- 自动切块检索
- 销售 / 客服直接问产品、交期、认证、MOQ、质保
- 输出可追溯来源

---

### 第 3 名：报价单 / BOM / 规格书解析 Agent
**适合度：8.0 / 10**

它很值得做，但更适合你在拿下前两个样板后再做。

原因：

- 和询盘转单天然相邻
- 客户价值高
- 但 PDF / Excel / 图片差异大，边界情况多
- 要想做得稳定，通常要补 OCR、表格解析、字段映射、人工校对

建议定位：

- 第二阶段加成模块
- 不是第一个独立产品

---

### 第 4 名：多语种售前 / 售后客服 Agent
**适合度：6.8 / 10**

这不是没需求，而是**太拥挤**。

问题在于：

- 大厂和成熟客服 SaaS 已经很强
- 客户对“客服系统”天然要求高：稳定性、渠道完整性、会话记录、质检、工单、权限、报表
- 售后还会被拉进很多灰区问题：退款、物流、赔付、纠纷、情绪安抚
- 一旦做成“你来兜底”，交付成本会非常高

如果你做它，建议只切一个窄点：

- “知识库先答 + 人工分流”
- 而不要卖“全自动客服平台”

---

### 第 5 名：工厂 SOP / 异常处理助手
**适合度：6.5 / 10**

它有长期价值，但更适合你有行业资源以后再切。

难点：

- 工厂现场流程差异大
- 要求知识准确且能落到责任人、升级路径、停线风险
- 决策链更长，导入周期更长
- 很容易被拉到“MES / 工业软件改造”这种重项目

最适合的切法：

- 先做“轻 SOP 问答助手”
- 只给出标准作业路径与升级路径
- 暂时不碰产线控制、不碰设备直连

## 最适合你的产品组合

不要一开始做 2 个完全独立产品。

建议你做成一个总品牌：

**AI 销售自动化工作台**

里面先放 2 个付费模块：

### 模块 A：Inquiry-to-Order Agent
- 统一接入询盘
- 打分
- 提取规格
- 首轮回信
- 指派销售

### 模块 B：Sales Knowledge Copilot
- 产品资料入库
- 销售 / 客服问答
- 认证 / MOQ / 交期 / 质保回答
- 回复素材建议

这样做的好处：

- 共享客户、共享文档、共享模型、共享后台
- 先卖询盘，再顺带卖知识库
- 或先卖知识库，再切询盘自动化

## MVP 架构

### 1. 接入层
- 官网表单
- 邮箱转发
- 网站聊天 webhook
- 后续接企业微信 / 飞书

### 2. 数据层
- SQLite / Postgres
- 询盘表
- 文档表
- 文档切块表
- 跟进动作表

### 3. 检索层
- MVP：BM25 / 关键词检索
- 第二阶段：Embedding + Rerank

### 4. Agent 层
- Lead Qualification Agent
- Spec Extraction Agent
- Reply Draft Agent
- Knowledge Retrieval Agent

### 5. 人工协同层
- 销售确认
- 一键复制回复
- 标记成交 / 丢单 / 待补资料

### 6. 输出层
- 英文首轮回复
- 销售待办
- 线索优先级
- 客户问答结果

## 你应该怎么卖

### 第一批客户画像

优先找这 4 类：

1. 深圳做独立站或平台招商的中小卖家
2. 电子、智能硬件、家居、配件类工贸一体企业
3. 外贸业务员 3-20 人的小团队
4. 有官网表单 / 邮箱询盘，但没有自动化分流能力的公司

### 第一句销售话术

不要说“我做 AI Agent 平台”。

直接说：

- “我帮你把官网、邮箱和聊天里的询盘自动筛客、自动回第一封英文邮件。”
- “我帮你把产品手册和 FAQ 变成销售能直接用的 AI 助手。”

这样客户能秒懂。

## 30 天落地节奏

### 第 1 周
- 做演示站
- 完成询盘表单接口
- 完成知识库上传
- 完成 mock 演示链路

### 第 2 周
- 接真实模型
- 跑通英文首轮回复
- 打磨 lead score 规则
- 输出 3 个行业 demo 数据

### 第 3 周
- 做 3 家试点企业
- 每家只接 1 个入口
- 记录真实问题和误判

### 第 4 周
- 做报价与页面包装
- 固化交付清单
- 做案例页和标准 demo

## 定价建议

### 入门版
- 2980 - 5980 元 / 月
- 单入口
- 单知识库
- 单业务负责人

### 标准版
- 8000 - 15000 元 / 月
- 多入口
- 多业务员分配
- 跟进统计
- 定制回复模板

### 定制版
- 2 万起
- 接 CRM / ERP / 企业微信 / 飞书
- 报价流程
- 私有部署或专属规则
